## 1. Introduction
This test plan covers the QA strategy for testing the Mobile Application (Android).  
The goal is to validate functionality, usability, performance, and accessibility.

---

## 2. Scope

### In Scope
- Login & Signup
- Home Screen
- Profile
- Notifications
- Settings
- Offline behavior
- Accessibility validation
- API validations using Postman

### Out of Scope
- Backend database testing
- Load testing beyond 5000 users
- Localization testing

---

## 3. Testing Types
- Functional Testing  
- Regression Testing  
- Smoke Testing  
- UI/UX Testing  
- Mobile Testing (Android)  
- API Testing  
- Accessibility Testing (WCAG 2.1)  

---

## 4. Test Approach
### Manual Testing
- Write detailed test cases
- Execute test cases on real devices/emulators
- Log bugs with severity & priority

### Automation Testing
- Selenium for web modules  
- Appium for mobile modules  
- TestNG framework  

---

## 5. Entry Criteria
- Build available in QA environment  
- API endpoints stable  
- Test data prepared

---

## 6. Exit Criteria
- All critical & major bugs resolved  
- Regression passed  
- No open blocker issues  

---

## 7. Deliverables
- Test Plan  
- Test Cases  
- Bug Reports  
- Regression Suite  
- Automation Scripts  
- Test Summary Report  

---

## 8. Tools Used
- Jira  
- Appium  
- Selenium  
- TestNG  
- Postman  
- Android Accessibility Scanner  

---

## 9. Team & Responsibilities
- **QA Engineer:** Mani (Test creation, execution, automation)
- **Developer:** App team  
- **PM:** Manages releases  

---

## 10. Risks
- Delayed builds
- Unstable APIs
- Limited devices for testing

---

## 11. Approval
- QA Lead  
- Project Manager
